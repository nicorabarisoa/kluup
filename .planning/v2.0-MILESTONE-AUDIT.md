## Cross-Phase Integration

### BLOCKER-1 — Landing page OAuth bypasses /auth/callback route
**Affects:** AUTH-01, AUTH-02, AUTH-03  
**File:** `app/page.tsx:112`

```js
// BROKEN — current code
redirectTo: typeof window !== 'undefined' ? window.location.href : undefined

// FIX — use auth/callback pattern (already correct on join/page.tsx:74 and game/page.tsx:1440)
redirectTo: `${window.location.origin}/auth/callback?next=${encodeURIComponent('/')}`
```

Sign-in from the landing page fails silently — PKCE code expires unexchanged, no session cookie is set, `user` is always null for landing-initiated sign-ins. Hosts who sign in from the landing page cannot have their `user_id` stored on their player row, and their stats are never auto-saved.

### BLOCKER-2 — Lobby page missing TOKEN_REFRESHED Realtime auth refresh
**Affects:** AUTH-02  
**File:** `app/room/[code]/lobby/page.tsx`

`game/page.tsx:1883` already has the correct pattern:
```js
if (event === 'TOKEN_REFRESHED' && session) {
  await supabase.realtime.setAuth(session.access_token)
}
```

The lobby page has no `onAuthStateChange` listener at all. A signed-in host waiting >1h in the lobby (common for large groups assembling) loses Realtime sync when their JWT expires — player join/leave events and room updates stop arriving silently.
