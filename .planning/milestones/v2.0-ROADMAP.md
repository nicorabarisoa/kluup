# Milestone v2.0: Auth & Stats

**Status:** ✅ SHIPPED 2026-06-12
**Phases:** 2–5
**Total Plans:** 21 (across 4 phases)

## Overview

Optional Google OAuth accounts with cross-session stats persistence. Any player or host can sign in with Google; their identity is linked to their player row and their session stats (designation count, confessions, volunteers, group titles) are saved to a personal profile. The anonymous game flow is structurally unchanged — accounts are never required to play.

## Milestone Summary

**Key Decisions:**

- `@supabase/ssr` for server-side auth (getUser, not getSession) — authoritative validation
- `session_uuid` generated in lobby `startGame()`, not in `makeInitialGameState` — ensures fresh UUID on replay
- `players.user_id` nullable FK — anonymous rows have null; no game flow change required
- IDEN-02 guard: localStorage pid takes precedence over user_id lookup (prevents cross-device ghost)
- `onConflict: ignoreDuplicates` on stats upsert — idempotent writes on replay/refresh
- PendingStatsFlusher as globally-mounted component in layout — stats survive room lifecycle independently
- `status='ended'` 30-min TTL exemption — defense-in-depth, not primary mechanism

**Issues Resolved:**

- OAuth PKCE `?code=<uuid>` collision with room code on `/join` — fixed via `/auth/callback?next=` pattern
- Silent stats loss when solo room swept during slow Google sign-in — fixed via localStorage stash + PendingStatsFlusher
- Landing page OAuth bypass (redirectTo: window.location.href) — fixed at audit close
- Lobby TOKEN_REFRESHED missing — fixed at audit close

**Tech Debt Carried:**

- Phases 02/03/05 verification `human_needed` (live smoke required) — confirmed by user at milestone close
- Phase 04 code review items (CR-01 stale effect, WR-01 no onAuthStateChange on join/landing) — non-blocking
- Profile page hardcoded French 'manches' string (WR-04) — i18n fix deferred to v3.0
- 21-archetype selection logic absent from profile page — irrelevant until tag_scores go non-zero in v3.0
