## v1 Requirements

### Health Check

- [x] **HLT-01**: Load balancer can `GET /api/health` and receive HTTP 200 with body `{"status":"ok","uptime":<seconds>}` — ✓ Phase 1 (pre-v2.0)
- [x] **HLT-02**: `uptime` is a non-negative integer representing seconds elapsed since the Next.js server process started — ✓ Phase 1 (pre-v2.0)

## v2 Requirements

### Authentication

- [x] **AUTH-01**: User can sign in with Google via OAuth (optional — game is fully playable without an account) — ✓ Phase 4 (join/page.tsx + landing) + BLOCKER-1 fixed at audit close
- [x] **AUTH-02**: Auth session persists across browser refresh and page navigation — ✓ Phase 2 (middleware JWT refresh) + BLOCKER-2 fixed at audit close; smoke test confirmed by user
- [x] **AUTH-03**: Signed-in user can sign out from any page — ✓ Phase 4
- [x] **AUTH-04**: Full anonymous game flow works without regression after every phase touching RLS or auth — ✓ Phase 2; smoke test run and confirmed by user at milestone close

### Player Identity

- [x] **IDEN-01**: `players` table has a nullable `user_id` FK — anonymous players have null — ✓ Phase 2; live DB migration confirmed by user
- [x] **IDEN-02**: A signed-in user joining a room on a new device is recognized via `user_id` lookup without creating a duplicate row — ✓ Phase 4

### Stats Persistence

- [x] **STAT-01**: At game end, each signed-in player's stats for that session are written to their account — ✓ Phase 5
- [x] **STAT-02**: Each session is stored as a separate history row — ✓ Phase 5
- [x] **STAT-03**: Stats writes are idempotent — `UNIQUE(user_id, session_id)` + `ignoreDuplicates` — ✓ Phase 5

### Stats Profile

- [x] **PROF-01**: Signed-in user can view their stats history on a `/profile` page — ✓ Phase 5
- [x] **PROF-02**: Anonymous users see a "sign in to save your stats" CTA on the end screen; stats saved after OAuth even with slow sign-in — ✓ Phase 5 (PendingStatsFlusher gap closure)
