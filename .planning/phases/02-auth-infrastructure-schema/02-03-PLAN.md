<objective>
Add a `session_uuid` field to the `GameState` type (D-01), initialize it to an empty string in `makeInitialGameState()`, and populate it with `crypto.randomUUID()` inside the lobby's `startGame()` before the room is written to the DB. This is the client-side identifier that Phase 4 will store as `session_id` in `user_session_stats`. Because `startGame()` already rebuilds `game_state` from scratch on every launch (including replay), each game session gets a fresh UUID automatically.

Purpose: Establish the per-session identity (D-01) that links a played game to its stats row in Phase 4, with zero new DB columns. Supports IDEN-01's broader identity infrastructure.
Output: One field added to a type, one initializer line, one assignment line — three small additive edits.
