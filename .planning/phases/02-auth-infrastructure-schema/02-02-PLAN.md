<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → middleware (every request) | Untrusted cookies arrive on every navigation; middleware must validate the session server-side, not trust the cookie blindly |
| OAuth provider redirect → /auth/callback | The `?code` param is attacker-influenceable; only Supabase Auth can complete a valid exchange |
| anonymous request → middleware | Anonymous traffic (no session cookie) must pass through with no error and no redirect |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-04 | Spoofing | session cookie validity | high | mitigate | Middleware uses `getUser()` (authoritative server-side validation against the Auth server), NOT `getSession()` (reads storage without validation). @supabase/ssr stores tokens in HttpOnly cookies inaccessible to JS. |
| T-02-05 | Elevation of Privilege | /auth/callback code exchange | high | mitigate | `exchangeCodeForSession` is enforced by Supabase Auth as single-use with a 5-minute TTL; PKCE verifier is stored HttpOnly by @supabase/ssr. A replayed or forged code fails the exchange and the handler silently redirects to / (no error surface). |
| T-02-06 | Denial of Service | anonymous flow regression via middleware | high | mitigate | `getUser()` returns `{ user: null }` with no error for anon requests — no redirect, request passes through. Matcher excludes static assets so middleware does not add latency to those. Full AUTH-04 smoke test runs in plan 02-04. Realtime uses WebSockets (outside the middleware chain) so game sync is unaffected. |
| T-02-07 | Information Disclosure | OAuth error details | low | mitigate | Per D-02, callback logs `error.message` server-side only and always redirects to `/` — no OAuth error detail is exposed to the client. |
| T-02-SC | Tampering | npm install (@supabase/ssr) | low | accept | RESEARCH.md Package Legitimacy Audit verdict is [OK] (official Supabase org, slopcheck OK, no postinstall script). No blocking-human checkpoint required. |
</threat_model>
