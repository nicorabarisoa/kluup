<objective>
Surface signed-in status in the in-game and lobby headers as an ultra-minimal green dot on the Quit button (D-06). Read auth state once per page root via `supabase.auth.getUser()`, derive a boolean `isSignedIn`, and conditionally render a 6px green dot. No text, no name, no tooltip. The anonymous flow is untouched — no dot, identical layout.

Purpose: Give signed-in users a passive confirmation their account is active during a session without adding any sign-in/sign-out UI to the game (D-01 forbids auth buttons in game/lobby/end). Supports AUTH-01 by making the optional signed-in state visible.
Output: `app/room/[code]/game/page.tsx` and `app/room/[code]/lobby/page.tsx` render a conditional green dot on their Quit buttons.
</objective>
