## Spacing Scale

Multiples of 4 only. All card layout uses explicit px inline styles (modern-screenshot discipline).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, color dot size (8px = 2×xs) |
| sm | 8px | Pill gaps, trait bar bottom margin, award row gap |
| md | 16px | Card internal column gap (matches existing ShareCard `gap: 16`) |
| lg | 24px | Card section top padding, flip affordance bottom margin |
| xl | 32px | Card horizontal padding (matches existing `padding: '28px 32px'`) |
| 2xl | 48px | End-screen section separation |
| 3xl | 64px | Not used in this phase |

**Phase-specific exceptions:**

- Card capture container: **always explicit px** — no `%`-based widths inside the 540×540 `div`. Computed as `style={{ width: Npx }}`.
- Touch target for tap-to-flip: minimum **44×44px** hit area on the card wrapper (iOS HIG minimum).
- Top color bar on card: **6px** height (matches existing ShareCard pattern exactly).
- Trait bar track height: **8px**. Max bar width: **160px** (computed as `Math.round(pct / 100 * 160)` px for capture).
- Award emoji container: **32×32px** fixed (consistent alignment regardless of emoji width).

## Card Architecture — 2-Face Contract

### Face state

```
activeCard: 'group' | 'personal'   (React state, default: 'group')
```

- **Only the active face is rendered** inside the capture `<div>` (P-07 prevention). Switching `activeCard` swaps the entire child tree.
- The capture `<div>` is **540×540px** with `position: 'relative'`, `overflow: 'hidden'`. Identical dimensions to existing ShareCard.

### Face 1 — Group (default)

Renders when `activeCard === 'group'`.

```
┌── 540px ───────────────────────────────────────┐
│  [6px color bar — theme color]                 │
│  ────────────────────────────────────────────  │
│  [Theme name]              [Klu·up logo]       │  ← 28px top, 32px sides
│  ────────────────────────────────────────────  │
│  "Ce soir vous étiez…" (14px muted)            │
│  [GROUP TITLE 42px display 800]                │  ← margin-top 6px
│  ────────────────────────────────────────────  │
│  [Duo awards block — surface bg #1A1A1A]       │
│  │ "Les duos de la soirée" (12px muted ALLCAPS)│
│  │ 🧲  Magnétisme Suspicieux                   │  ← 14px, weight 500
│  │     Nico & Sarah           (14px muted)     │
│  │ 🧠  Même longueur d'onde                    │
│  │     Léa & Thomas           (14px muted)     │
│  │ ⚔️   Les Ennemis Jurés                       │
│  │     Jules & Paul           (14px muted)     │
│  │ 🔥  Les Complices                           │
│  │     Nico & Léa             (14px muted)     │
│  ────────────────────────────────────────────  │
│  [spacer — flex: 1]                            │
│  [Player pills]                                │
│  kluup.app (13px faint, centered)              │
└────────────────────────────────────────────────┘
```

**Edge case — fewer than 2 qualified awards (D-03):** The duo awards block is omitted entirely. Face 1 shows group title + spacer + player pills. No "no duos" placeholder text is shown; the card is simply the group title card as before.

**Duo awards block layout:**
- Container: `background: C.surface`, `borderRadius: 18`, `padding: '14px 18px'`, `borderLeft: '4px solid ${meta.color}'` (matches existing "Moment fort" block style)
- Section label: `font-size: 11px`, uppercase, letter-spacing `0.06em`, color `C.muted`, margin-bottom `10px`
- Award rows: `gap: 8px` between rows, `display: flex`, `alignItems: 'flex-start'`
- Emoji cell: `width: 32px`, `flexShrink: 0`, `fontSize: 18px`
- Award name: `fontSize: 14px`, `fontWeight: 600`, `color: C.text`
- Player names: `fontSize: 13px`, `color: C.muted`, on the same line after `·` separator — format: `{Pseudo1} & {Pseudo2}`

### Face 2 — Personal (archetype + existing stats)

Renders when `activeCard === 'personal'`.

```
┌── 540px ───────────────────────────────────────┐
│  [6px color bar — theme color]                 │
│  ────────────────────────────────────────────  │
│  [Theme name]              [Klu·up logo]       │
│  ────────────────────────────────────────────  │
│  [Moment fort block — existing, unchanged]     │
│  ────────────────────────────────────────────  │
│  [Personal stats pills — existing, unchanged]  │
│  ────────────────────────────────────────────  │
│  [Archetype block — surface bg #1A1A1A]        │
│  │ "Ton archétype" (11px muted ALLCAPS)        │
│  │ LE FARCEUR  (28px Bricolage 800, UPPERCASE) │
│  │ ────────────────                            │
│  │ [trait 1 colored label] ████████░░  45%     │
│  │ [trait 2 colored label] ██████░░░░  30%     │
│  │ [trait 3 colored label] ███░░░░░░░  25%     │
│  ────────────────────────────────────────────  │
│  [spacer]                                      │
│  [Player pills — existing]                     │
│  kluup.app                                     │
└────────────────────────────────────────────────┘
```

**Archetype block layout:**
- Container: same style as duo awards block (`background: C.surface`, `borderRadius: 18`, `padding: '14px 18px'`, `borderLeft: '4px solid ${meta.color}'`)
- Section label: `fr.archetypes.card_title` — 11px muted ALLCAPS letter-spacing 0.06em
- Archetype name: 28px Bricolage Grotesque weight 800, `C.text`, `textTransform: 'uppercase'`, margin-top 6px, margin-bottom 12px
- Divider: 1px `C.border`, margin-bottom 10px
- Trait rows: top 3 only. Each row: `display: flex`, `alignItems: 'center'`, `gap: 8px`, margin-bottom 8px (last row 0)
  - Trait label: `fontSize: 13px`, `fontWeight: 600`, `color: TRAIT_COLORS[traitKey]`, `width: 88px`, `flexShrink: 0`
  - Bar track: `height: 8px`, `flex: 1`, `background: C.border`, `borderRadius: 4`
  - Bar fill: `height: 8px`, `background: TRAIT_COLORS[traitKey]`, `borderRadius: 4`, `width: barWidthPx` (EXPLICIT PIXELS: `Math.round(pct / 100 * 160)`)
  - Percent: `fontSize: 12px`, `color: C.muted`, `width: 36px`, `textAlign: 'right'`, `flexShrink: 0`

**Edge case — total points = 0 (D-03):** Render the archetype block with the `fr.archetypes.archetype_fallback` string ("Une simple personne") in place of the archetype name. Trait rows are omitted (show nothing below the name). No "not enough data" variant — fallback is always shown when points > 0 is false.

**Edge case — fewer than 3 traits with points > 0:** Render only the traits that have points > 0 (minimum 1, maximum 3). Never render a 0-point trait bar.

## Flip Interaction Contract

### Tap-to-flip (D-01)

- **Trigger:** single tap/click anywhere on the card `<div>` (the 540-visual-width container in the live DOM view, not the capture container).
- **State change:** `activeCard` toggles between `'group'` and `'personal'`.
- **Animation:** CSS `opacity` fade — 150ms ease-out fade to 0, swap content, 150ms ease-in fade to 1. No 3D CSS `rotateY` (WebKit backface-visibility quirk — D-04 / P-07).
- **No swipe gesture** — tap only (avoids scroll conflict on mobile long card).

### Flip affordance (D-01 — Claude's discretion)

Displayed below the card content, inside the card wrapper but outside the capture container:

```
↻ tap to see your archetype    (Face 1)
↻ tap to see the group         (Face 2)
```

- Position: centered, below the card's bottom edge, 8px gap
- Style: 12px DM Sans weight 400, color `C.faint` (`#555555`), `cursor: pointer`
- i18n keys to add: `card.flip_to_personal` / `card.flip_to_group` (see Copywriting section)
- The affordance is rendered in the live DOM only — **never** inside the capture container

### Share button (D-02)

- Position: below the flip affordance, outside the flip surface
- Label: `fr.end.share_cta` (existing: "Partager la soirée" / "Share the night")
- Exports: `domToBlob(captureRef.current, { width: 540, height: 540, scale: 2 })` — only the currently-active face
- The capture `<div>` is a separate hidden `<div ref={captureRef}>` that renders only the active face (no flip UI, no affordance text)

## Capture Container Contract (modern-screenshot safety)

Enforced by P-07 and D-04:

| Rule | Value | Rationale |
|------|-------|-----------|
| Container dimensions | 540×540px, explicit inline `width`/`height` | Matches existing `domToBlob` call — do not change |
| Scale | 2 (1080px output) | Retina quality — matches existing |
| Background | `C.bg` = `#0D0D0D` | Explicit `backgroundColor` on `domToBlob` options |
| Trait bar widths | `Math.round(pct / 100 * 160)` px, applied as `style={{ width: barWidthPx }}` | Never `%` in capture context |
| Face rendering | Render ONLY the active face inside capture container | No `display:none` hidden face — swap entire subtree |
| Font await | `await document.fonts.ready` before `domToBlob` | Existing pattern — do not remove |
| Emoji in awards | Retain emoji (🧲🧠⚔️🔥) — existing Android/WebKit risk accepted | Test on real device before ship. Fallback: replace with Unicode text symbols if glitches found. |
| Capture trigger | `exportCard()` reads `captureRef.current` (the off-screen capture div, not the visible card) | Prevents transform:scale measurement error |
