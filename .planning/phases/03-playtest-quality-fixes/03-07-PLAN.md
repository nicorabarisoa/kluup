<objective>
Fix SC-4 (real local gap): after quitting from the lobby, returning to `/join?code=XXXX` shows an empty pseudo field. Root cause (diagnosed): the pre-fill is sourced ENTIRELY from `getPlayerId(code)` â†’ a `players.pseudo` row lookup, but `onQuit` calls `clearPlayerId(code)` AND deletes the player row, destroying both inputs. The pseudo text is never persisted independently.

Fix: persist the last-used pseudo independently of the player id/row in a separate localStorage key (`kluup_pseudo_<CODE>`), written on successful join. Read it as a FALLBACK in the `/join` pre-fill effect when `getPlayerId` is null or the row lookup returns nothing. `clearPlayerId` (called on quit) must NOT clear this key. The input stays editable and joining still requires an explicit submit.

Purpose: SC-4 â€” a player rejoining after quitting gets their old pseudo pre-filled (editable, explicit submit), per the locked UAT decision.
Output: edits to `lib/utils.ts` (new helpers) and `app/join/page.tsx` (write on join + fallback read). All user-facing text via i18n. `tsc --noEmit` + `next build` green.
</objective>
