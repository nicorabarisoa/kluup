# Phase 3 — UI Design Contract: Playtest Quality Fixes

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase nature:** Bug-fix + minor UX. No new visual concepts. All tokens, components, and layout patterns are inherited from the existing MVP codebase. This contract documents the constraints and adds the three new UI elements introduced in this phase: lobby quit button, pseudo-taken error state, and mid-round join toast.

## Spacing Scale

Declared values (must be multiples of 4). Source: observed in lobby, game, and join pages.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding |
| sm | 8px | Compact element spacing, gap between inline items |
| md | 16px | Default horizontal page padding, element gaps |
| lg | 24px | Section vertical padding |
| xl | 32px | Major vertical gaps between sections |
| 2xl | 48px | Top/bottom section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets (buttons in RoundHeader): minimum 32px height (`h-8`) for compact header controls
- Primary CTA buttons: 16px vertical padding (`py-4`) = 56px effective height
- Footer safe area: 32px bottom padding (`pb-8`) to clear mobile home bar
- Max content width: 448px (`max-w-md`) for game/lobby columns; 340px for hero form; 320px (`max-w-xs`) for join form

| Accent type-B | `--color-accent-b` | `#7B2FFF` | Type B game rounds only |

Accent (`#FF3C6F`) reserved for:
- Primary CTA button background (`startGame`, `joinRoom`)
- Kluup logo accent (second syllable "up")
- Host badge ring / highlight

- Active/selected state on theme cards (inverted: white background, `#0D0D0D` text)

**Error state color:** Muted red inline text — use `#FF3C6F` at reduced opacity (`#FF3C6F` with `opacity-80` or `color: '#FF6B8A'`) for inline validation errors. Must NOT use a full red background block — inline text under the input field only.

## New UI Elements — Phase 3 Specs

### 1. Lobby Quit Button (D-13)

**Location:** `app/room/[code]/lobby/page.tsx` — top-right area of the centered column, adjacent to or replacing the current `LangSwitch`-only header row.

**Visual spec:**
- Exact same styling as the quit button in `RoundHeader` in `app/room/[code]/game/page.tsx`
- `px-3 h-8 rounded-xl text-xs font-medium`
- Background: `#1A1A1A` (`C.surface`), Border: `1px solid #252525` (`C.border`), Color: `#888888` (`C.muted`)
- Font: `var(--font-body)`
- Label: `fr.game.quit` (existing i18n key — "Quitter" / "Leave" / "Salir" / "Verlassen")
- Position: top-left of the header row (mirroring RoundHeader layout: quit on the left)
- The existing `LangSwitch` stays on the right

**Behavior spec:**
- Same quit logic as in-game: host transfer to oldest remaining player if others present; room deletion (cascade) if last player
- No confirmation dialog in lobby (the game already skips dialog for simplicity; the lobby quit should match — single tap, immediate action)
- After quit: redirect to `/` (home)

**Source:** `app/room/[code]/game/page.tsx` lines 130–139 (RoundHeader quit button pattern)

---

### 2. Pseudo Already Taken Error (D-01, D-02, D-03)

**Location:** `app/join/page.tsx` — below the pseudo input field.

**Visual spec:**
- Inline error text, NOT an alert/modal
- Position: immediately below the `<input>` for pseudo (between input and join button)
- Font: DM Sans, 12px (`text-xs`), weight 400
- Color: `#FF3C6F` at 80% opacity — use inline style `color: 'rgba(255,60,111,0.85)'` or a slightly lighter `#FF6B8A` to distinguish from primary CTA
- No background, no border, no icon — plain text only
- Appears only when server returns a unique-constraint conflict (23505 on pseudo); hidden otherwise (conditional render, not `visibility:hidden`)
- Text: `fr.join.pseudo_taken` (new i18n key — see Copywriting Contract below)
- Does not replace existing `room_not_found` or `join_error` alerts — those remain as `alert()` calls for now

**Interaction spec:**
- Error clears when the user types in the pseudo input (`onChange`)
- Error clears when the user changes the room code
- Submit button remains enabled during error state (user can retype and resubmit)

---

### 3. Mid-Round Join Toast Notification (D-09, D-10)

**Location:** Overlay in `app/room/[code]/game/page.tsx` — positioned at the top of the viewport, centered.

**Visual spec:**
- A floating pill/badge, NOT a full-width banner
- Dimensions: `auto width`, `px-4 py-2`, `rounded-2xl` (matching app's border-radius language)
- Background: `#1A1A1A` (`C.surface`), Border: `1px solid #252525` (`C.border`)
- Font: DM Sans, 14px, weight 500 (`font-medium`)
- Color: `#FFFFFF`
- Position: `fixed top-4 left-1/2 -translate-x-1/2 z-50` — above all game content
- Max width: 280px; text truncated with ellipsis if pseudo is very long
- Entry animation: fade in (`opacity 0→1`, 150ms ease-out)
- Exit animation: fade out (`opacity 1→0`, 150ms ease-in) before unmounting
- Auto-dismiss: after **2.5 seconds** (within the 2–3s range specified in D-10 / CONTEXT.md)

**Content spec:**
- Text: `fr.game.player_joined(pseudo)` — new i18n key (see Copywriting Contract)
- Example: "Nico a rejoint la partie" / "Nico joined the game"

**Behavior spec:**
- Triggered by receiving a `player_joined` broadcast event on channel `votes-broadcast-${id}`
- At most one toast visible at a time; if two players join in quick succession, queue them — show the second after the first dismisses (or replace immediately if simpler; Claude's discretion)
- Toast does NOT appear in lobby (lobby already shows live player list)
- Toast does NOT appear on the end screen

**Implementation pattern:** Inline state `[toastMessage, setToastMessage]` in the game page component; `useEffect` on broadcast events sets the message and sets a `setTimeout` to clear it after 2500ms.

### 4. Pre-Populated Pseudo on Rejoin (D-14)

**Location:** `app/join/page.tsx` — the pseudo `<input>` field.

**Visual spec:**
- Same input field, same styling (`rounded-2xl px-4 py-4 text-white text-base outline-none`)
- When a previous pseudo is found in localStorage for the given room code, pre-populate `value` with that pseudo
- The field remains fully editable — no visual indication that the value was auto-filled (no badge, no lock icon)
- Optional subtle hint text adjacent to the field: `fr.join.pseudo_prefilled_hint` — see Copywriting Contract. If used, render as 11px muted text (`color: #888`, `text-xs`) below the input, above any error message. This is Claude's discretion — implement only if it reduces confusion.

**Interaction spec:**
- The user MUST submit (tap "Rejoindre") — the pseudo is not silently reused
- If the user clears the field and submits empty, existing disabled-state behavior blocks submission
- The room code field is also pre-populated (already implemented via `searchParams.get('code')`)

### 5. Landing Page Copy Fix (D-15)

**Location:** `app/page.tsx` — the `fr.landing.players_hint` i18n key (and equivalents in EN/ES/DE).

**Scope:** String-only change, no visual modifications.

**Before/After:**
- FR: `"3 à 10 joueurs · 10–20 min"` → `"Conseillé entre 3 et 10 joueurs · 10–20 min"`
- EN: `"3 to 10 players · 10–20 min"` → `"Recommended 3–10 players · 10–20 min"`
- ES: `"3 a 10 jugadores · 10–20 min"` → `"Recomendado entre 3 y 10 jugadores · 10–20 min"`
- DE: `"3 bis 10 Spieler · 10–20 Min"` → `"Empfohlen 3–10 Spieler · 10–20 Min"`

**No layout changes.** The text renders at existing size (`text-xs`, `#555555`).

## Copywriting Contract

All new keys must be added to **all 4 language dictionaries** in `lib/i18n.ts` (FR, EN, ES, DE). The `Dict` type enforces key exhaustiveness at compile time.

| Element | Key path | FR | EN | ES | DE |
|---------|----------|----|----|----|-----|
| Pseudo taken error | `join.pseudo_taken` | "Ce pseudo est déjà pris, choisis-en un autre." | "That name is already taken — pick another one." | "Ese nombre ya está en uso, elige otro." | "Dieser Name ist schon vergeben, wähle einen anderen." |
| Mid-round join toast | `game.player_joined` (function: `(pseudo: string) => string`) | `(p) => \`${p} a rejoint la partie\`` | `(p) => \`${p} joined the game\`` | `(p) => \`${p} se unió a la partida\`` | `(p) => \`${p} ist beigetreten\`` |
| Landing players hint (updated) | `landing.players_hint` | "Conseillé entre 3 et 10 joueurs · 10–20 min" | "Recommended 3–10 players · 10–20 min" | "Recomendado entre 3 y 10 jugadores · 10–20 min" | "Empfohlen 3–10 Spieler · 10–20 Min" |
| Lobby quit label | `game.quit` | Already exists: "Quitter" | "Leave" | "Salir" | "Verlassen" |
| Rejoin hint (optional) | `join.pseudo_prefilled_hint` | "Ton ancien pseudo est pré-rempli." | "Your previous name has been pre-filled." | "Tu nombre anterior está rellenado." | "Dein vorheriger Name wurde vorausgefüllt." |

**Primary CTA for this phase:** "Rejoindre" (join flow) / "Lancer la partie" (lobby). No new CTAs introduced.

**Empty state:** Not applicable — this phase adds no empty-list UI.

**Destructive action — Lobby Quit:**
- No confirmation dialog (matching in-game quit behavior for consistency)
- Single tap = immediate quit
- Rationale: the lobby is pre-game; the cost of accidental quit is lower than in-game, and the in-game button already has no modal

**Error states:**
- Pseudo taken: inline text below pseudo input, 12px, `rgba(255,60,111,0.85)` — message: `fr.join.pseudo_taken`
- Room not found: existing `alert()` — no change
- Network error on join: existing `alert()` — no change

- All touch targets: minimum 32px height maintained (matching existing pattern)
