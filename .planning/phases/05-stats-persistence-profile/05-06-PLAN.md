<objective>
Defense-in-depth for the same UAT gap as plan 05-05 (fix direction 2 from `.planning/debug/oauth-return-lands-on-home.md`): exempt `status='ended'` rooms from the aggressive ~90s `cleanup_dead_rooms()` sweep so a just-finished game survives a multi-minute OAuth round-trip. Today `cleanup_dead_rooms()` (`supabase/lifecycle.sql` Block 3) deletes ANY room with `last_activity < now() - interval '90 seconds'` with no exemption for ended games; when the sole player of a solo room leaves for Google, the heartbeat stops and the room (with its `game_state.session_uuid` and `votes`) is gone in ~60-150s.

Plan 05-05 already makes stats survival independent of room lifetime (the localStorage stash). This plan complements it: keeping ended rooms alive for ~30 min also lets the existing in-EndScreen retroactive save path work on return, and lets "Rejouer" (replay from the same lobby) still function after a sign-in detour. It is NOT a substitute for 05-05 (a user can always exceed any TTL) — both ship together.

Purpose: an `ended` room represents a completed game whose end screen the user may still be interacting with (sharing the card, signing in). It should not be swept on the same 90s clock as an abandoned empty lobby.
Output: `supabase/lifecycle.sql` Block 3 updated with a status-aware TTL; the user applies it manually to the prod DB (Supabase SQL editor).
</objective>
