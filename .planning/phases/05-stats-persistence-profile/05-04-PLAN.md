<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous visitor → /profile route | Unauthenticated access must not reveal any stats |
| Browser client → user_session_stats SELECT | A signed-in client must read ONLY its own rows (no IDOR via user_id param) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-12 | Elevation of Privilege | unauthenticated /profile access | mitigate | Client guard: `getUser()` at mount → `router.push('/')` when null; RLS `stats_select_own` is the second layer (a query without an auth session returns 0 rows) |
| T-05-13 | Information Disclosure | IDOR — reading another user's stats | mitigate | Fetch carries no user_id parameter from the URL; RLS `stats_select_own` (`auth.uid() = user_id`) scopes every SELECT to the caller server-side; optional `.eq('user_id', user.id)` is defense-in-depth, never a trust boundary |
| T-05-14 | Spoofing | name-chip navigation target | accept | `/profile` is a same-origin static route; no user-controlled redirect target |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; all APIs from the existing `@supabase/supabase-js` + `next` dependencies (RESEARCH audit) |
</threat_model>
